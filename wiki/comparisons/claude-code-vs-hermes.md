# Claude Code vs Hermes Agent

Claude Code 和 [[hermes-agent]] 都可用于驱动本地文件与知识库工作流，但它们在使用重心上不同。当前知识库主要基于教程和体验型来源，因此该页面适合当作方向判断，而不是精确产品规格说明。 ^[raw/articles/youtube-obsidian-second-brain-tutorial-2026-06-30.md] ^[raw/articles/hermes-agent-orange-book-2026-06-09.md] ^[raw/articles/claude-obsidian-second-brain-2026-04-14.md]

| 维度 | Claude Code | Hermes Agent |
|---|---|---|
| 核心印象 | 面向本地文件/项目的强交互代理 | 更通用的 Agent 平台，强调工具、记忆、技能和多渠道 |
| 与 Obsidian 关系 | 常直接围绕 vault 工作 | 更适合作为 vault 之外的执行层 |
| 典型场景 | 本地整理、生成页面、项目内协作 | 自动化工作流、跨平台分发、长期可复用流程 |
| 学习曲线 | 对本地 CLI / 工程环境更友好 | 对想要“一个能逐步成长的同事型 Agent”的用户更有吸引力 |

## 粗略选择建议

- 如果目标是围绕本地 vault、项目文件和即时整理展开，优先考虑 [[claude-code]]。 ^[raw/articles/claude-obsidian-second-brain-2026-04-14.md] ^[raw/articles/youtube-obsidian-second-brain-tutorial-2026-06-30.md]
- 如果目标是把知识整理、消息分发、持续监控和多步骤自动化串成长期系统，优先考虑 [[hermes-agent]]。 ^[raw/articles/hermes-agent-guide-2026-04-15.md] ^[raw/articles/hermes-agent-orange-book-2026-06-09.md] ^[raw/articles/youtube-obsidian-second-brain-tutorial-2026-06-30.md]
- 实际上，两者也可以并存：一个偏“本地编辑与交互”，一个偏“通用执行与自动化编排”。
