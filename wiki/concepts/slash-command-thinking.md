# Slash Command Thinking

Slash Command Thinking 指的是把一类可重复的思考或整理动作抽象成简短命令入口，例如通过类似 `/save`、`/autoresearch` 或用户自定义斜杠命令，把复杂流程压缩成稳定调用方式。它的价值不在命令本身，而在于让“触发工作流”比“重新写一段长提示词”更轻。 ^[raw/articles/claude-obsidian-second-brain-2026-04-14.md] ^[raw/articles/youtube-obsidian-second-brain-tutorial-2026-06-30.md]

## 在知识库工作流中的意义

- 降低非技术用户使用 AI 工作流的门槛。 ^[raw/articles/youtube-obsidian-second-brain-tutorial-2026-06-30.md]
- 让日记提炼、资料归档、研究整理等动作形成固定入口。
- 便于把 [[obsidian-ai-workflows]] 变成一套可重复使用的方法，而不是一次性灵感。
